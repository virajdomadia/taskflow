# TASKFLOW — CANONICAL PRODUCT & MARKETING CONTEXT

## 1. PRODUCT DEFINITION

**Product Name:** TaskFlow
**Category:** Task Management SaaS
**Audience:** Small teams (2–15 people) who are overwhelmed by complex tools (Asana, Notion) or messy workflows (Slack, spreadsheets, docs)

**Core Problem:**
Work is scattered across multiple tools, leading to:
* unclear ownership
* lost context
* missed deadlines
* constant status-check interruptions

**Root Cause:**
Lack of clarity and accountability—not just “too many tools”

**Core Value Proposition:**
TaskFlow provides one simple, focused system to plan, assign, and track work with full visibility—without the complexity of traditional project management tools.

**Positioning:**
A lightweight, clarity-first alternative to bloated, over-engineered tools.

**DO NOT POSITION AS:**
* enterprise project management software
* feature-heavy or highly customizable platform
* “all-in-one workspace” (too generic)

---

## 2. CORE MESSAGING

**Primary Promise:**
“Task management that isn’t overcomplicated”

**Key Differentiator:**
Simplicity + clarity + fast adoption

**Competitor Framing:**
* Asana/ClickUp → too complex, heavy
* Notion → flexible but becomes chaotic
* Spreadsheets/Slack → unstructured, fragmented

**TaskFlow = simple + structured + focused**

---

## 3. HERO COPY (DEFAULT)

**Headline:**
Finally, Task Management That Isn’t Overcomplicated

**Subheadline:**
TaskFlow gives small teams a clear, simple way to plan and track work—without the clutter of tools like Asana or Notion.

**Primary CTA:**
Start Free — No Setup Needed

**Secondary CTA:**
Watch Demo

---

## 4. FEATURE PRINCIPLES (NOT JUST FEATURES)

All features must communicate BENEFITS, not mechanics.

**Core Benefits:**
* Clear ownership → no dropped tasks
* Real-time visibility → no status chasing
* Context in one place → no searching across tools
* Simple structure → no onboarding friction
* Built for small teams → no unnecessary complexity
* Reduced tool overload → fewer systems to manage

**Rule:**
If a feature sounds technical, rewrite it as a user outcome.

---

## 5. SOCIAL PROOF STYLE

**Tone:**
* grounded
* specific
* non-generic

**Avoid:**
* exaggerated claims
* vague praise (“amazing tool”)

**Preferred Structure:**
[Name] — [Role]
Specific before/after transformation

**Example Pattern:**
“Before [old tool/process], we struggled with [problem]. TaskFlow helped us [specific improvement].”

---

## 6. PRICING STRUCTURE

**Starter (Free):**
* Up to 5 users
* Core task management
* Kanban view
* Basic collaboration

**Growth ($8/user/month):**
* Unlimited users
* Timeline & calendar views
* File sharing & comments
* Integrations (Slack, Google Drive)

**Scale ($15/user/month):**
* Automation
* Advanced insights
* Permissions & admin controls
* Priority support

**Rules:**
* Keep pricing simple and transparent
* Highlight Growth as “Most Popular”

---

## 7. LANDING PAGE STRUCTURE (MANDATORY ORDER)

1. **Hero** (clear positioning + CTA)
2. **Problem** (pain amplification)
3. **Solution** (3 pillars only)
4. **Features** (6-card grid, scannable)
5. **Differentiation** (vs complex tools)
6. **Testimonials** (3 max)
7. **How It Works** (3 steps)
8. **Pricing** (3 tiers)
9. **FAQ** (5–6 objections)
10. **Final CTA** (strong close)

---

## 8. DESIGN PRINCIPLES

**Visual Style:**
* minimal
* clean
* low cognitive load

**UI Requirement:**
The product UI shown MUST reinforce simplicity.
If UI looks complex → messaging fails.

**Avoid:**
* cluttered dashboards
* dense text blocks
* too many colors or elements

---

## 9. COPY RULES (STRICT)

**DO:**
* write at 6–8th grade readability
* prioritize clarity over cleverness
* keep sentences short
* make benefits obvious in <3 seconds

**DO NOT:**
* use buzzwords (synergy, optimize, leverage)
* over-explain features
* sound like enterprise software
* be vague

**Test:**
If copy could apply to any SaaS → rewrite it.

---

## 10. DIFFERENTIATION GUARDRAILS

Every output MUST reinforce:
**“Simple vs Complex Tools”**

If this contrast is missing:
→ output is considered weak

---

## 11. SUCCESS CRITERIA

Any generated asset (copy/design/code) should pass:
1. Can a user understand the product in 10 seconds?
2. Is the simplicity claim believable?
3. Is TaskFlow clearly different from Asana/Notion?
4. Does it reduce perceived effort to start?

If NOT:
→ iterate until it does

---

## 12. DEFAULT TONE
* clear
* confident
* practical
* no hype

Write like explaining to a smart, busy teammate—not pitching to investors.
